# Behaviors

## Purpose

Cross-cutting application policies implemented as pipeline behaviors (commonly with MediatR).

## Typical contents

- Validation behavior
- Logging/metrics abstraction behavior (if you choose to do it here)
- Idempotency behavior (application-level policy)

## Out of scope

- Framework-specific middleware
- Infrastructure logging implementations
